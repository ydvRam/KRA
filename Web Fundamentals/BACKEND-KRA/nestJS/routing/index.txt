1️⃣ Request Flow in NestJS

When a client sends a request:

Client Request
      ↓
Route Params / Query / Body
      ↓
Pipes (ValidationPipe)
      ↓
DTO Validation (class-validator)
      ↓
Controller Method
      ↓
Service Logic
      ↓
Response

If something fails → Exception Filters handle errors.

2️⃣ Route Parameters

Route parameters are dynamic values inside the URL path.

Example request:

GET /users/5

Here 5 is a route param.

Controller Example
import { Controller, Get, Param } from '@nestjs/common';

@Controller('users')
export class UsersController {

  @Get(':id')
  findUser(@Param('id') id: string) {
    return `User id is ${id}`;
  }

}
Request
GET /users/10
Response
User id is 10
3️⃣ Query Parameters

Query parameters come after ? in the URL.

Example:

GET /users?page=1&limit=10
Controller Example
import { Query } from '@nestjs/common';

@Get()
findUsers(
  @Query('page') page: string,
  @Query('limit') limit: string
) {
  return {
    page,
    limit
  };
}
Request
GET /users?page=2&limit=5
Response
{
  "page": "2",
  "limit": "5"
}
4️⃣ Request Body

Body is used mainly for POST / PATCH requests.

Example request:

POST /users

Body:

{
  "name": "Ram",
  "email": "ram@gmail.com"
}
Controller Example
import { Body } from '@nestjs/common';

@Post()
createUser(@Body() body: any) {
  return body;
}

But in real NestJS apps we use DTOs instead of any.

5️⃣ DTO (Data Transfer Object)

DTO defines the structure of request data.

Example DTO:

export class CreateUserDto {

  name: string;

  email: string;

}

Controller:

@Post()
createUser(@Body() dto: CreateUserDto) {
  return dto;
}

DTO makes the code typed and structured.

6️⃣ class-validator

The library used for validating request data.

Install it:

npm install class-validator class-transformer
DTO with Validation
import { IsString, IsEmail } from 'class-validator';

export class CreateUserDto {

  @IsString()
  name: string;

  @IsEmail()
  email: string;

}

Now Nest checks:

name must be string
email must be valid email
Example Invalid Request
POST /users
{
  "name": 123,
  "email": "abc"
}

Response:

{
 "statusCode": 400,
 "message": [
   "name must be a string",
   "email must be an email"
 ],
 "error": "Bad Request"
}
7️⃣ Common Validation Decorators
Decorator	Purpose
@IsString()	value must be string
@IsEmail()	valid email
@IsOptional()	field not required
@IsNumber()	number validation
@IsBoolean()	boolean value
@MinLength()	minimum characters

Example:

import { IsString, IsEmail, IsOptional } from 'class-validator';

export class CreateUserDto {

  @IsString()
  name: string;

  @IsEmail()
  email: string;

  @IsOptional()
  age?: number;

}
8️⃣ class-transformer

Used to transform incoming data into class objects.

Example:

Request:

GET /users?id=5

Normally:

id = "5" (string)

But using transformer:

id = 5 (number)

Example:

import { Type } from 'class-transformer';

export class QueryDto {

  @Type(() => Number)
  page: number;

}
9️⃣ ValidationPipe

ValidationPipe tells Nest to validate DTO automatically.

Without it, validators won't run.

Add in main.ts.

import { ValidationPipe } from '@nestjs/common';

app.useGlobalPipes(new ValidationPipe());

Now every request body using DTO will be validated.

Best Production Version
app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
    forbidNonWhitelisted: true,
    transform: true,
  }),
);

Options meaning:

Option	Purpose
whitelist	removes unknown fields
forbidNonWhitelisted	throws error if unknown field sent
transform	converts types automatically

Example:

Request:

{
 "name":"Ram",
 "email":"ram@mail.com",
 "hack":"value"
}

If whitelist:true → hack removed.

🔟 HttpException

Used to manually throw HTTP errors.

Example:

import { HttpException, HttpStatus } from '@nestjs/common';

throw new HttpException(
  'User not found',
  HttpStatus.NOT_FOUND
);

Response:

{
 "statusCode": 404,
 "message": "User not found"
}
Nest has built-in exceptions

Instead of HttpException, we usually use:

Exception	Status
NotFoundException	404
BadRequestException	400
UnauthorizedException	401
ForbiddenException	403

Example:

import { NotFoundException } from '@nestjs/common';

throw new NotFoundException('User not found');
1️⃣1️⃣ Exception Filters

Exception filters control how errors are returned to the client.

Default Nest response:

{
 "statusCode": 404,
 "message": "User not found",
 "error": "Not Found"
}

But companies often create custom error format.

Example filter:

import { Catch, ExceptionFilter, ArgumentsHost } from '@nestjs/common';

@Catch()
export class AllExceptionsFilter implements ExceptionFilter {

  catch(exception: any, host: ArgumentsHost) {

    const ctx = host.switchToHttp();
    const response = ctx.getResponse();

    response.status(500).json({
      message: exception.message
    });

  }

}

Register globally:

app.useGlobalFilters(new AllExceptionsFilter());

  getUser(@Param('id') id: string) {
    return { id };
  }

  getUsers(@Query('page') page: number) {
    return { page };

  @Post()

  }

}
1️⃣3️⃣ Final Request Flow
POST /users
        ↓
Body parsed
        ↓
DTO created
        ↓
ValidationPipe runs
        ↓
class-validator checks rules
        ↓
Controller method executes
        ↓
Response sent

If validation fails:

Validation error
       ↓
HttpException
       ↓
Exception filter
       ↓
Error response
